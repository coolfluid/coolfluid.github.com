Detailed Description

Adds fonctionnalities to Property class.

An option is a piece of data of which user can modify the value. An option provides:

  • a value (set to a default value when the option is built)
  • a description string
  • basic/advanced modes
  • change value using a XML node
  • restricted list of values
  • triggers management By default, an option is advanced. It can be marked as basic by calling mark_basic() method.
    Developer can define a restricted list of values, meaning that only those values are the valid. The list is obtained with restricted_list() method. Values can be added using Boost.Assign library. Note that this list always contains at least one value: the default one.
    Triggers are functions called whenever the value is modified. They can be registered with attach_trigger() method. Triggers take no parameter and return nothing.

    About the name:
    Although the option name is stored as a string so that there is no language limitation to its format, developers are asked to respect some rules in order to make the API learning easier:
  • name should be in lower case
  • if the name is composed of multiple words, use undescores ('_') to separate them (no spaces).
  • name should be explicit and as short as possible. Use acronyms if possible. An option has also a "pretty name". This is a human readable form of the name. It is intended to be displayed on the user interfaces. It should contain spaces between words and a capital letter at least for the first word. Note that most OS guidelines recommand this kind of strings to have a captial letter for each word and to only contain verbs if necessary. Pretty name is optional and can be empty.
