Coolfluid 3
A Collaborative Simulation Environment

ESA Summer of Code in Space Ideas page

Should our organization be accepted, we will offer students the opportunity to work on our team. Even though the Coolfluid 3 framework lives in the context of scientific computing, we do aim to provide an end-user friendly application, which implies that many kinds of tasks need to be performed:

  • Python interface strengthening
  • Improving visualization support
  • Improve file format support
  • Structured and unstructured grid generation
  • Numerical model development

Potential students are free to contact us with their own project proposals, or may use the (non-exhaustive) list below to find inspiration. Any project proposal must be well documented and show that some studying of the code was done before finalizing the application. We encourage you to discuss project ideas early on the mailing list or through the github issue system.

Students applying to our organization will benefit from working in a small, dynamic team, with a wealth of scientific and non-scientific programming experience. Through the collaborating institutions, access is available to computing resources, allowing for example scalability testing on a computing cluster environment.

Plasma flow simulation

Within the developer institutions of Coolfluid3, between the Royal Military Academy of Belgium and the von Karman Institute a strong cooperation is established in order to develop time-accurate, incompressible fluid flow solver by means of stabilized finite element method. A new branch of this cooperation is to investigate the unsteady/turbulent behavior of plasma flows. Better understanding in this field by means of computational simulations are believed to help the design of experimental facilities, which are the primary tools for ground testing thermal protection system materials. The task of the student is to develop a tool which allows the investigation of the unsteady behavior of plasma torches, based on the already existing flow solver. The student can make extensive use of our Embedded Domain Specific Language to speed up the development of the solver.

The Project will consist of the following tasks:

  • Literature review: understanding the physical model and selecting validation testcase(s).
  • Learning and understanding the structure of our software, in emphasis on embedded domain specific language and Boost::Proto.
  • Development of the plasma flow solver both for 2D, 2D axis-symmetric and 3D cases.
  • Performing validation (comparison against results available in literature)
  • Preparation of unit and performance tests.

References

  • Current code for our incompressible Navier-Stokes solver
  • Paper about the domain specific language
  • Thesis describing the flow physics

Project category

This project requires understanding of the finite element method and preferably experience in fluid simulation. C++ experience is beneficial, but model development is simplified by the use of a domain specific language embedded in C++, avoiding low-level C++ work.

Extending visualization support

We currently support writing out basic ParaView files directly. The current writer is not complete, since surface patches are not exported yet, for example. A first step could be the completion of this writer, making use of the VTK library. Full support for our native mesh format could be attained by making use of the ExodusII writer from VTK. It would be nice to have more interactive visualization perhaps using the VTK interface directly. This would for example allow writing out contour plots as the simulation is running. A very interesting extension would be the ability to directly interface with pvserver, allowing interactive online parallel viewing while the simulation runs on a cluster.

As a minimum, the student will develop a fully functional writer/reader using the VTK library.

References

A first implementation of a VTK plugin for coolfluid can be found here. It is currently only used for mesh tetrahedralization.

Project category

This project requires some C++ experience, and the ability and willingness to learn external APIs, in this case VTK and ParaView. Prior knowledge of numerical simulation is not required. The student can adapt the difficulty: getting the file exporter to work is the first step. After that, depending on student interest and ability, a reader could be implemented or more direct interfacing with VTK/Paraview could be investigated.

Mentors

Bart Janssens

Structured mesh generation

A fully parallel structured grid generator exists in the code base. The current implementation only supports structured blocks containing straight edges. A natural extension would be to support 3D NURBS blocks, enabling structured grid generation around curved surfaces such as an airfoil.

Project category

Basic C++ project, requiring some study of NURBS basics. No special use of external libraries is required.

Mentors
